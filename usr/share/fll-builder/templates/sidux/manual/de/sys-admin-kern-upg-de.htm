Neue Kernel installieren

Im linken Menü auf der sidux-Webseite findet man den aktuellen Kernel in drei Varianten:

  • up - Kernel für die i386 Prozessor-Familie
  • smp - Kernel für die i386 Prozessor-Familie mit dual core
  • 64-smp - Kernel für die AMD64 Prozessor-Familie, wenn 64 bit sidux benutzt wird

Den Kernel für den eigenen Prozessor downloaden (wir nennen dieses Archiv hier kernel.zip, es wird aber immer einen anderen Namen haben - z. B. kernel-2.6.17.7-slh-up-1.zip )

Ein Terminal öffnen, in das Verzeichnis, wo man das heruntergeladene Archiv gespeichert hat, wechseln und das Archiv entpacken:

$ unzip kernel.zip -d kernel-current
In das erzeugte Verzeichnis kernel-current wechseln:: 
$ cd kernel-current

Die nächsten Schritte benötigen Rootrechte, also:

$ sux
# ./install-kernel-sidux.sh

WICHTIG: wenn bestimmte Treibermodule benötigt werden, z. B. für wlan, muss man die entsprechenden deb-packages installieren, so z. B. falls man madwifi benutzt, dann muss man das entsprechende deb-paket nachinstallieren.

Jetzt den PC neu starten. Der neue Kernel wird erst bei einem Neustart automatisch gewählt. Bei Problemen kann man hier auch einen älteren Kernel auswählen.

Wenn man keinen ATI- oder NVIDIA-Grafiktreiber benötigt, passt man die Lautstärke der Soundkarte an:

alsaconf

Im Anschluss wird mit kmix oder alsamixergui die gewünschte Standardlautstärke gewählt und gespeichert mit:

alsactl store

, und das Update des Kernels ist beendet

Falls man eine ATI- oder NVIDIA-Graphikkarte hat, sollte der erste Reboot nach einem Kernel-Update mit Runlevel 3 gemacht werden (dies erreicht man, indem 3 in der Grub-Boot-Zeile angehängt wird.

Darauf achten, dass der aktuellste gcc-Compiler installiert ist - man kann dies mit folgendem Befehl herausfinden:

# apt-cache policy $(dpkg -l |grep -v base |grep -e
'gcc-'|awk '{print $2;}' |tail -1)

Falls notwendig, bitte die neueste Version von gcc mittels apt-get installieren, und - falls gewünscht - die aktuellen Grafikkartentreiber zur 3D-Unterstützung installieren (hier unsere Infoseite zur Installation proprietärer Grafiktreiber im Handbuch).

Nach erfolgreicher Installation kann man die entpackten Kernel-Pakete (und das Kernel-Archiv) löschen.

Alte Kernel entfernen:

Nach der erfolgreichen Installation können alle alten Kernel aus dem System entfernt werden. Dafür kann man das Skript kernel-remover benutzen, welches sich im Paket sidux-kernelhacking befindet. Dieses Skript ermöglicht eine Auswahl der Kernel, welche entfernt werden sollen.

su -
kernel-remover
