3D-Treiber für nVidia-Grafikkarten

Im Augenblick gibt es drei Methoden 3D-Treiber für nVidia-Grafikkarten zu installieren:

Methode 1: install-sidux-binary-gfx.sh

In einer Konsole folgendes Skript als Root anwenden:

install-sidux-binary-gfx.sh

Das Skript wählt automatisch den korrekten Treiber für deine Karte, installiert ihn und ändert die xorg.conf entsprechend. Damit der Treiber verwendet wird muss X neu gestartet werden (Ctrl+Alt+Backspace).

Methode 2: !h2

install-nvidia-debian.sh herunterladen:

cd /usr/local/bin ; wget -Nc http://techpatterns.com/nv
chmod +x install-nvidia-debian.sh

ODER: verwende h2-fixes-du.sh um die Grafiktreiber zu installieren: du-fixes-h2.sh. Dann, um nur 3D Treiber zu installieren:

du-fixes-h2.sh -kiwdt

Nativer Treiber "nv" von X.Org für nVidia-Grafikkarten

Dieser X-Server-Treiber von X.Org (siehe xserver-xorg für eine eingehendere Beschreibung) bietet Unterstützung für die NVIDIA Riva-, TNT-, GeForce- und Quadro-Karten.

Er wird installiert mit:

Aus KDE Abmelden mit Ctrl+Alt+F1.
Anmelden als root.
Dann: 
# init 3 
ausführen.
(Eine Nachricht sollte anzeigen, dass der X-Server angehalten wurde.)
apt-get update
apt-get install xserver-xorg-video-nv
init 5 && exit

Anmerkung: Dies ist nicht der binäre NVidia-Treiber, welcher 3D-Unterstützung bietet. Der binäre NVidia-Treiber ist aber ausschließlich binär zu erhalten und im Grunde nicht von Linux unterstützt.

Nun muss die /etc/xorg.conf editiert werden. Du solltest zuvor eine Sicherungskopie erstellen, fälschliches Editieren der /etc/xorg.conf wird dir sonst viele Tränen und eine zeitaufwendige Fehlersuche bringen.

Um /etc/xorg.conf zu editieren, kannst Du sie, Rootrechte vorrausgesetzt, per Rechtsklick öffnen. Im Abschnitt SECTION DEVICE den Display-Treiber in nv ändern.

Editieren der xorg.conf geschieht auf eigenes Risiko.

Mehr Informationen über X.Org

HD NTFS-Partitionen mit ntfs-3g beschreibbar machen

Achtung: ntfs-3g ist ein früher Betatreiber. Nicht benutzen ohne vorher Daten zu sichern, vor allem nicht auf Arbeitsplattformen! Benutzung auf eigenes Risiko, Datenverlust nicht ausgeschlossen!

In eine Konsole das folgendes eingeben: Siehe Partitionieren der Festplatte, um Benennungskonventionen der Festplatte in Linux nachzulesen.

umount /dev/XdXX
mount -o silent,umask=0,no_def_opts,allow_other -t ntfs-3g /dev/XdXX /media/XdXX
exit
exit

Die NTFS-Partition sollte jetzt mit Schreibzugriff eingehängt sein und man sollte Daten darauf speichern können. Ein letztes Mal: Sei gewarnt! Experimentell!
