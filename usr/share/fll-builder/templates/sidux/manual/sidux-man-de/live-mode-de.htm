Das root Passwort auf der Live-CD

Achtung: Wann immer man mit root Rechten arbeitet sollte man wissen was man macht. Für das Surfen im Internet sind keine root Rechte nötig

Auf der Live-CD ist kein root Passwort gesetzt. Es gibt mehrere Möglichkeiten ein Programm auszuführen:

  • Ein root Passwort setzen:

    Eine Konsole öffnen:

    sidux@0[sidux]$ sudo passwd dummy
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
sidux@0[sidux]$

    Dieses Passwort kann den Rest der Live Sitzung verwendet werden.

  • Das Programm, z.B. GParted unter Verwendung von su-me starten:

    in KDE: Alt+F2 drücken, in den Dialog:

    su-me && gparted

    eingeben

  • Das Programm aus einer root-shell starten:

    einen Terminal / eine Konsole öffnen

    sidux@0[sidux]$ su dummy
root@0[sidux]# gparted

Unionfs - Software im Live Modus installieren

Unionfs erzeugt ein überlagerndes Dateiensystem im Arbeitsspeicher das es ermöglicht im Live Modus apt-get zu verwenden oder Nvidia/Radeon Treiber zu installieren. siduxs Nvidia/Radeon Skripte aktivieren Unionfs im Live Modus automatisch. Ansonsten muss man Unionfs mit einem Cheatcode in der Boot Kommandozeile aktivieren. Hierzu fügt man der Zeile dies hinzu:

unionfs

Mit diesem Cheatcode werden Dateiänderungen in /tmp/unionfs gespeichert. Änderungen am Dateisystem gehen nach dem Herunterfahren verloren.

Wenn man den Unionfs Cheatcode beim Booten nicht verwendet hat, kann man ihn auch während der Livesitzung in der Konsole aktivieren.

sidux@0[sidux]$ sudo dummy fix-unionfs
activating Unionfs now.

Jetzt kann man mit apt-get z.B. zusätzliche Software installieren:

sidux@9[sidux]$ su dummy
root@0[sidux]# apt-get update
(...)
root@0[sidux]# apt-get install kdegames

Es gibt zwei Möglichkeiten Unionfs ein wenig permanenter auszuführen:

root0@[sidux]# unionrw

und

root0@[sidux]# unionro

Der unionrw Befehl kann auf unterschiedliche Weise genutzt werden:

  1. auf eine Partition auf der Festplatte verweisen
  2. auf eine loop Datei verweisen, wie eine ext2 loopback Datei
  3. auf ein Verzeichnis verweisen

All diese drei Wege ermöglichen es permanente Änderungen zu machen und sie zu speichern. Verwendet man denselben Cheatcode kann man eine eigene Programmauswahl wieder verfügbar machen. Das Heimverzeichnis /home wird jedoch hiermit nicht gespeichert, dazu muss man den home Cheat verwenden.

Nun zur dritten erwähnten Möglichkeit:

root0@[sidux]# unionro

Dies ist im Prinzip ähnlich unionrw, erzeugt aber einen read-only Zugang auf bereits vorher installierte Modifikationen. Man kann auch unionro mit einem neuen unionrw Cheat verbinden. Achtung: Diese Cheats sind experimentell, keiner von ihnen wird in der sidux Boothilfe erwähnt

NTFS-Partitionen mit ntfs-3g beschreibbar machen

Achtung: ntfs-3g ist ein früher Betatreiber. Nicht benutzen ohne vorher Daten zu sichern, vor allem nicht auf Arbeitsplattformen! Benutzung auf eigenes Risiko, Datenverlust nicht ausgeschlossen!

In eine Konsole das folgendes eingeben: Siehe Partitionieren der Festplatte um Benennungskonventionen der Festplatte in Linux nachzulesen.

sidux@9[sidux]$ su dummy
root@0[sidux]# fix-unionfs
root@0[sidux]# umount /dev/XdXX
root@0[sidux]# mount -o silent,umask=0,no_def_opts,allow_other -t ntfs-3g /dev/XdXX /media/XdXX

Das NTFS-Volume sollte jetzt mit Schreibzugriff eingehängt sein und man sollte Daten darauf speichern können. Ein letztes Mal: Sei gewarnt! Experimentell!
