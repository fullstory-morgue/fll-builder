Die sidux-CD herunterladen

Download-Links und Mirrors findet man auf unserer Homepage.

Mit Bittorrent herunterladen

Bittorent ist ein Filesharing Protokoll das auf der sidux-CD ist. Es ermöglicht es Dateien gleichzeitig herunter- und für andere heraufzuladen. Alles was man tun muss ist z.B.: 

# btdownloadcurses --max_upload_rate 12 ftp://dummy/sidux/files/sidux-XXXX/sidux-XX.iso.torrent

In der Konsole sollte das Herunterladen bald beginnen. Achtung: In dem Verzeichnis aus dem der Befehl ausgeführt wird muss genug Platz frei sein, also ca. 700 MB. Nach dem Beenden des Herunterladens bitte das Programm weiterlaufen lassen, so dass andere die Datei von dir herunterladen können. Das wird unsere Server entlasten und das Herunterladen für andere schneller machen. Eine gute Einleitung zum Thema gibts hier. Brians BitTorrent FAQ and Guide

Bitte nicht vergessen die MD5-Summe zu prüfen.

Sollte jemand einen FTP-Server mit gengend Traffic zur Verfügung stellen können, liebend gern einen Administrator kontaktieren. Alternativ kann man auch im Forum oder dem IRC-Channel seine Absicht bekanntgeben.

MD5-Summe und Validation von heruntergeladenen Dateien

Das ist die Check-Summe einer Datei. Diese Check-Summe wird zur Integritätsprüfung dieser Datei benutzt. Dabei wird die momentane MD5-Summe der Datei mit einer bekannten früheren Summe verglichen. So kann festgestellt werden, ob die Datei verändert oder beschädigt wurde. Dies ist bei heruntergeladenen Datein ratsam.

Wenn die MD5-Summe der heruntergeladenen Datei mit der Summe die in der MD5-Datei angegeben ist üereinstimmt, ist sicher, dass die Datei korrekt heruntergeladen wurde. Unter Linux erhält man die MD5-Summe einer Datei mit:

$ md5sum zu_prüfende_datei

das dauert ein wenig, die Summe wird dann auf der Konsole ausgegeben. Mit dem md5summer (486 kb) kann die MD5-Summe auch unter Windows geprüft werden.

Die ISO-Images von sidux werden immer mit der entsprechende MD5-Summe zum Download angeboten und sollten vor dem Brennen immer geprüft werden.Dies sollte in einem Terminal durchgeführt werden. Man wechselt in das Verzeichnis in der sich die Iso-Datei und die MD5-Summe sich befinden und kann (zB) mit dem Befehl:

$ md5sum -c sidux-Name.iso.md5

die MD5-Summe checken lassen. Wenn sie nicht übereinstimmen erhält man vom Programm die Meldung:

sidux-Name.iso: Fehlschlag md5sum: Warnung: 1 von 1 berechneten Prüfsumme stimmen nicht überein.

Ist aber die heruntergeladene Datei korrekt, dann beendet sich das Programme ohne jegliche Meldung.

Eine Live-CD mit Windows brennen

Selbstverständlich kann das ISO-File auch unter Windows gebrannt werden. Die geladene Datei muß als CD-Abbild gebrannt werden. Die ISO-Datei keinesfalls als Datei auf die CD brennen. Wenn z.B. Winrar mit der Endung .iso verknüpft ist, irritiert das viele Einsteiger da angenommen wird, es handelt sich um ein Archiv. Aus der ISO Datei muß eine CD erstellt werden. Nicht die ISO-Datei auf die CD brennen! Ebenso ist es zwingend notwendig das Abbild als "Disk-At-Once" zu brennen.

Die folgende Anleitung beschreibt das Erstellen der CD mit Nero (nicht nero express). Danke an Schwaller, der das beschrieben hat.

Die heruntergeladenene ISO-Datei auswählen und "öffnen" anklicken.

Die Cd mit Linux Brennen

Wer bereits Linux auf dem Rechner hat, kann die CD mit jedem installierten Brennprogramm erstellen. Auf sidux ist z.B. K3b, dort im Menüpunkt "Extras" -> "CD-Abbilddatei brennen..." anklicken, das zu brennende ISO-File (z.B. sidux-Name.iso) auswählen, DAO brennen - Disk At Once.

K3B berechnet zuerst die MD5-Summe des ISO-Files (dauert einen Moment). Stimmt die angezeigte Prüfsumme mit der angegebenen Zeichenfolge der sich im selben Ordner befindlichen MD5-Datei (z.B. sidux-Name.iso.md5) überein, war der Download erfolgreich und das File kann mit einem Klick auf "Start" gebrannt werden. Oder auf das Symbol neben der kalkulierten MD5-Summe klicken um manuell auf die gegebene Summen-Datei zu verweisen.

Direkt auf der Shell brennt man mit:

# cdrecord dev=0,0,0 driveropts=burnfree -dao -overburn -v sidux-200X-0X-chaos.iso

Möchte man auf eine CDRW brennen kann man diese löschen mit

# cdrecord dev=0,0,0 blank=fast

Die Werte für dev=x,x,x (im Beispiel 0,0,0) werden von cdrecord so ausgegeben:

# cdrecord --scanbus

oder für ATAPI-Brenner:

# cdrecord --scanbus dev=ATAPI

Es gibt auch eine Handbuchseite über das Brennen auf der Shell.

I/O Fehler beim Brennen mit Linux

Mit dem Kernel >=2.6.8 läuft cdrecord mit root-Rechten. Deswegen funktioniert der Aufruf mit suid root aus Sicherheitsgründen nicht mehr

Lösung: Benutze den K3b-Einrichtungs-Assistent oder:

#chmod -s /usr/bin/cdrecord*

Eine aktuelle sidux-CD unter Verwendung von XDELTA-Patch herstellen

xdelta ist ein Programm, das unter anderem aus einem ISO-Image und einer Differenzdatei ein neues ISO-Image erstellt. Es ist also wie geschaffen, kleinere Fehlerkorrekturen, Fixes, ohne große Downloadkosten zu erlauben.

So sind die Differenzdateien von RC18 nach RC20, die Fixe PATCH-RC18-RC19.XDELTA und PATCH-RC18-RC19.XDELTA nur 16,5 MB bzw. 5,4 MB groß, während das ISO selbst 553 MB groß ist.

Voraussetzung:

  • ein ISO-Abbild auf der Festplatte, z.B. sidux.iso
  • Eine Differenzdatei, z.B. PATCH-RC19-RC20.XDELTA

Beide Dateien müssen im gleichen Ordner sein.

Erstellen, als User in der Konsole:

$ cd /pfad/zum/iso
$ xdelta patch PATCH-RC19-RC20.XDELTA

Das Programm extrahiert die notwendigen Dateinamen aus dem PATCH. Es führt auch eine md5sum-Überprüfung durch. Achtung! Es prüft nicht ob genug Speicherplatz vorhanden ist.
